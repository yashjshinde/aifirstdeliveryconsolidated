# d365-ce

> **Fat agent** owning every Power Platform sub-domain on the Customer Engagement side: model-driven CE, Canvas apps, Power Pages, PCF controls, all Power Automate (cloud + desktop), plus plugins / JS web resources / business rules / BPF / classic workflows / BPM. Multi-file sub-platform FDD/TDD per [ADR-0005](../../design/adr/0005-d365-ce-multi-file-sub-platform.md). Domain-scoped per [ADR-0006](../../design/adr/0006-doc-scope-domain-vs-feature.md).

## What

The d365-ce agent produces specs, plans, FDD, TDD, blueprint, test-plan, and per-feature outputs for everything on the CE / Power Platform side. The FDD and TDD use a multi-file sub-platform pack — `/fdd` assembles `_index.template.md` (SW Phoenix shape) with the sub-platform files (model-driven / canvas / power-pages / pcf / power-automate) selected per `project.config.yaml` scope. Form mockups are rendered to interactive HTML via the form-mockup helper (PORTED from SW source per ADR-0005).

This agent does NOT own F&O work (`d365-fo`), Azure-side integration (`integration`), Power BI / CE SSRS reporting (`reporting`), cross-agent architecture (`solution-architect`), effort estimation (`solution-estimate`), or ALM round-trip (`alm`).

## How

- **New CE feature** — `/spec --feature <slug> --source fresh` → `/review --approve` → `/plan` → `/clarify --approve` → `/task` → `/validate --approve` → `/implement` → `/document` → `/alm-extract`
- **Add to existing domain doc** — `/fdd` (or `/tdd` or `/blueprint`) appends a feature-tagged section to the domain doc + appends rows to table-shaped sections with the `feature-id` column. Idempotent: re-running on an unchanged feature is a no-op.
- **Form mockups** — generated by `/fdd` invoking the helper at `templates/fdd-helpers/form-mockup-generator.prompt.md` for every entity in scope. Outputs land at `projects/{p}/d365-ce/fdd-assets/mockups/{entity-form}.html` (organised by entity).
- **Cross-domain spec** — `/spec` → `/review --approve` flags integration / reporting concerns → `/split` emits handoffs to the affected agents (with skeleton specs in their feature folders).
- **Brownfield project** — when `project.config.yaml mode: brownfield`, `/impact` reads `agents/brownfield/_brownfield/inventory.json` (Phase 7) to surface existing CE customisations the new feature must coexist with.

## What this agent does NOT do

- Does NOT author the SW Phoenix FDD body content beyond R19 A1-A15 hooks. The full per-section content is queued as **backlog bk-005** for incremental authoring.
- Does NOT build Canvas / Pages / PCF / Power Automate sub-platform body content beyond the skeletons. That's **backlog bk-006 / bk-007**.
- Does NOT author the form-mockup helper from scratch — it's PORTED VERBATIM from the SW source.
- Does NOT push to ADO / JIRA. That's the `alm` agent.
- Does NOT estimate effort. That's `solution-estimate`.
