# Platform Rules — d365-fo (brownfield)

## Accepted input file types

Under `_brownfield/input/d365-fo/`:

| Input | Format | Notes |
|---|---|---|
| Deployable package | `.zip` | Contains compiled assemblies + metadata; analyse via `ModelUtil.exe -unpack` |
| Model file | `.axmodelfile` | Unpacked X++ source per model; preferred for full analysis |
| AOT XML | per-folder XML | `Tables/`, `Forms/`, `Classes/`, `Views/`, `Queries/`, `Workflows/`, `Reports/`, etc. |
| X++ source | `.xpp` / `.xml` | Preferred; enables full method-level analysis |
| ER configurations | `.xml` (per format/model/mapping) | Treated as schema-asset (format) or process-asset (mapping) |
| DMF projects | `.xml` (project definitions) | Treated as integration-asset |
| Retail extensions | per-folder XML under `RetailDevelopment/` | Module-gated: requires `module:retail` |

## Scan strategy

1. Unpack deployable package: `ModelUtil.exe -unpack` → `_brownfield/extracted/d365-fo/{model-name}/`
2. Walk `AxTable/*.xml` → tables (incl. table extensions distinguishable by suffix `.Extension`)
3. Walk `AxTableExtension/*.xml` → table extensions
4. Walk `AxClass/*.xml` → classes (incl. CoC extensions identified by `[ExtensionOf]` attribute in source)
5. Walk `AxForm/*.xml` + `AxFormExtension/*.xml` → forms
6. Walk `AxDataEntityView/*.xml` → data entities
7. Walk `AxCompositeDataEntityView/*.xml` → composite entities
8. Walk `AxAggregateMeasurement/*.xml` → aggregate measurements
9. Walk `AxKPI/*.xml` → KPIs
10. Walk `AxView/*.xml`, `AxQuery/*.xml` → views, queries
11. Walk `AxMenuItem*/*.xml` → menu items (Action / Display / Output)
12. Walk `AxWorkflow*/*.xml` → workflow templates / approvals / categories
13. Walk `AxServiceClass/*.xml` → service classes
14. Walk `AxBusinessEvent/*.xml` → business events
15. Walk `AxER*/*.xml` → ER formats / models / mappings
16. Walk `AxSecurityRole/*.xml`, `AxDuty/*.xml`, `AxPrivilege/*.xml`, `AxSecurityKey/*.xml` → security
17. Walk `AxBatchJob/*.xml` → batch jobs
18. Walk `AxNumberSequence/*.xml` → number sequences
19. Walk `AxReport*/*.xml` → F&O native SSRS reports (RDP-driven)
20. Walk retail folders only when `Retail` module declared in `Descriptor.xml`

## Module detection signals (F&O side)

- **retail**: `Descriptor.xml` lists the Retail module OR retail folders present in unpacked source
- **commerce**: similar to retail; check `RetailExtensions`
- **project-operations**: presence of `proj*` table prefixes

## Analysis rules

### Tables / Table extensions

- Documented depth: every field (count + name + EDT + mandatory + label)
- Indexes + Primary index identified
- Methods: list with signature; logic narrative when X++ source present
- Cross-refs: forms using this table; classes referencing this table; reports using this table; data entities exposing this table

### CoC extensions (Chain-of-Command)

- One doc per `[ExtensionOf]` decorated class
- Hooked-on type + method + replace/extend pattern
- Source order: when multiple CoC classes hook the same method, document the chain

### Data entities

- Documented depth: every field with mapping back to underlying table
- Key fields + composition pattern
- Used-by: DMF projects, virtual entities, business events

### F&O workflows

- Distinct from CE classic workflows and Power Automate; use F&O workflow framework
- Documented depth: stages + approvers + delegation rules + escalation

### Security keys / duties / privileges / roles

- Privilege → permission matrix (table + entry-point + access level)
- Duty → privileges roll-up
- Role → duties roll-up

### Business events

- Event class + payload schema + emitted-from table/class
- Subscribers: external recipients listed if endpoint config present

## Cross-references for FO (mechanically computed)

- table ↔ table-extension
- class ↔ CoC-extension (extension hooks on class)
- table ↔ form (form data sources)
- table ↔ data-entity (entity references table)
- data-entity ↔ DMF-project (project's entity list)
- workflow ↔ table (workflow runs on table)
- privilege ↔ duty ↔ role (rollup)
- batch-job ↔ class (batchable class)
