# Scan Template — d365-fo

## Pre-flight (`/prepare`)

- `_brownfield/input/d365-fo/` exists
- At least one `.axmodelfile` OR an unpacked AOT folder with `Descriptor.xml`

## Extraction walk (`/scan`)

### Step 1 — Unpack model(s)

For each `.axmodelfile`:
- Unpack: `ModelUtil.exe -unpack`
- Walk to `<model>/Descriptor.xml`; emit `fo-model` artifact

### Step 2 — Tables, table extensions, EDTs, base enums

```
walk: AxTable/*.xml             -> table
walk: AxTableExtension/*.xml    -> table-extension
walk: AxEdt/*.xml               -> edt
walk: AxEnum/*.xml              -> base-enum
walk: AxView/*.xml              -> view
walk: AxQuery/*.xml             -> query
```

For each table: extract Fields/AxTableField/* nodes (Name + ExtendedDataType + Mandatory).

### Step 3 — Classes (incl. CoC)

```
walk: AxClass/*.xml             -> class (with CoC detection)
walk: AxClass/*.xml > attributes -> mark as CoC when [ExtensionOf] attribute present
```

### Step 4 — Forms + Form extensions + Menu items

```
walk: AxForm/*.xml              -> fo-form
walk: AxFormExtension/*.xml     -> fo-form-extension
walk: AxMenuItemAction/*.xml    -> menu-item (Action)
walk: AxMenuItemDisplay/*.xml   -> menu-item (Display)
walk: AxMenuItemOutput/*.xml    -> menu-item (Output)
```

### Step 5 — Data entities, composite entities, aggregate measurements, KPIs

```
walk: AxDataEntityView/*.xml            -> data-entity
walk: AxCompositeDataEntityView/*.xml   -> composite-entity
walk: AxAggregateMeasurement/*.xml      -> aggregate-measurement
walk: AxKPI/*.xml                       -> kpi
```

### Step 6 — F&O workflows + service classes + business events

```
walk: AxWorkflowTemplate/*.xml          -> fo-workflow
walk: AxWorkflowCategory/*.xml          -> fo-workflow-category
walk: AxWorkflowApproval/*.xml          -> fo-workflow-approval
walk: AxServiceClass/*.xml              -> service-class
walk: AxBusinessEvents/*.xml            -> business-event
```

### Step 7 — Batch jobs + Number sequences + Print management

```
walk: AxBatchJob/*.xml              -> batch-job (when present)
walk: AxNumberSequence/*.xml        -> number-sequence
walk: AxPrintManagement*/*.xml      -> print-mgmt
```

### Step 8 — Electronic Reporting (ER)

```
walk: AxER*/*.xml
  AxERFormat/*    -> er-format
  AxERModel/*     -> er-model
  AxERMapping/*   -> er-mapping
```

### Step 9 — Security keys / duties / privileges / roles

```
walk: AxSecurityKey/*.xml       -> security-key
walk: AxPrivilege/*.xml         -> privilege
walk: AxDuty/*.xml              -> duty
walk: AxSecurityRole/*.xml      -> fo-role
```

### Step 10 — F&O native SSRS reports

```
walk: AxReport/*.xml            -> fo-ssrs-report (RDP-driven)
walk: AxRDP/*.xml               -> fo-rdp (cross-ref for report)
```

### Step 11 — Workspaces + Mobile workspaces + DMF projects

```
walk: AxWorkspace/*.xml             -> workspace
walk: AxMobileWorkspace/*.xml       -> mobile-workspace
walk: AxDMFProject/*.xml            -> dmf-project
```

### Step 12 — Retail-module-gated walk (when retail module detected)

```
walk: RetailDevelopment/*       -> retail-extension, channel, hardware-profile, payment-connector
```

Run module-detection AFTER step 1's `Descriptor.xml` parse and BEFORE step 12 to gate retail walks.

### Step 13 — Dual-write + Virtual entity providers

```
walk: AxDualWriteProjectConfig/*.xml    -> dual-write-config
walk: AxVirtualEntityProvider/*.xml     -> virtual-entity-provider
```

### Step 14 — Validate inventory coverage

`validate_inventory_coverage` against `inventory.platforms.d365-fo`.

## Output schema

Conforms to [`schemas/brownfield-inventory.v1.json`](../../../schemas/brownfield-inventory.v1.json).
