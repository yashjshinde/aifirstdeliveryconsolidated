# Scan Template — d365-ce

> Drives `/scan` for CE inputs. Each artifact-type below maps to one binding under [`templates/bindings/d365-ce/`](../bindings/d365-ce/). The scan walks accepted inputs (per [`constitution/platforms/d365-ce.md`](../../constitution/platforms/d365-ce.md)) and emits a flat list of instances per artifact-type into `inventory.json`.

## Pre-flight (`/prepare`)

- `_brownfield/input/d365-ce/` exists
- At least one `.zip` (solution export) OR a folder with `solution.xml` at the root

## Extraction walk (`/scan`)

Each step writes to `inventory.platforms.d365-ce.{artifactType}[]`.

### Step 1 — Unpack solution(s)

For each `.zip` under `_brownfield/input/d365-ce/`:
- Extract to `_brownfield/extracted/d365-ce/{solution-name}/`
- Record solution metadata from `solution.xml`: `UniqueName`, `Version`, `Managed`, `Publisher.UniqueName`, `Publisher.CustomizationPrefix`

Artifact-type emitted: `solution`, `publisher`.

### Step 2 — Entities + attributes + keys + relationships

```
walk: Entities/*/Entity.xml
emit: entity (one per file)
  fields: extract attributes/* nodes (Name, AttributeType, RequiredLevel)
  keys: extract EntityKeys/* and combine with Entity.xml > entitySetName
  relationships: extract OneToMany/* and ManyToOne/* per entity
```

Artifact-type emitted: `entity`. Module-detection signal: emit list of entity names for module-detector.

### Step 3 — Forms + Views + Charts + Dashboards

```
walk: Entities/<entity>/FormXml/forms/*/formxml.xml         -> form
walk: Entities/<entity>/SavedQueries/*/savedquery.xml       -> view
walk: Entities/<entity>/Charts/*/visualization.xml          -> chart
walk: Dashboards/*/formxml.xml                              -> dashboard
```

For each form: extract `<tabs>/<tab>/<sections>/<section>/<rows>/<row>/<cells>/<cell>` hierarchy.

### Step 4 — Plugin assemblies + steps

```
walk: PluginAssemblies/*/*.cs (when source present)         -> source for validate_plugin_logic
walk: SdkMessageProcessingSteps/*.xml                       -> plugin-step
walk: PluginTypes/*.xml                                     -> plugin (per class)
```

For each plugin step: extract `Message`, `PrimaryEntityName`, `Stage`, `Mode`, `Rank`, `FilteringAttributes`, `Images/*`.

### Step 5 — Workflows + BPFs + Business Rules

```
walk: Workflows/*.xml
  if <Category>0</Category>  -> classic-workflow
  if <Category>4</Category>  -> bpf
  if <Category>2</Category>  -> business-rule
```

### Step 6 — JavaScript + HTML + XML + image web resources

```
walk: WebResources/*
  by extension: .js -> js-function (extract per top-level function); .html -> html-web-resource; .xml -> xml-web-resource; image -> image-web-resource
```

### Step 7 — Security roles + Teams + FLS profiles

```
walk: Roles/*.xml                                           -> security-role
walk: Teams/*.xml                                           -> team
walk: FieldSecurityProfiles/*.xml                           -> fls-profile
walk: HierarchyRules/*.xml                                  -> hierarchy-security
```

### Step 8 — Reference / config data

```
walk: TransformationConfigurations/*.xml                    -> dup-detection-rule
walk: AppModules/*.xml                                      -> model-driven-app
walk: SiteMap.xml                                           -> sitemap (one per app)
walk: EnvVariables/*.xml                                    -> env-variable
walk: ConnectionReferences/*.xml                            -> connection-reference
walk: Templates/*.xml                                       -> email-template / word-template / excel-template (by content-type)
walk: RibbonDiffXml/*.xml                                   -> ribbon-diff (one per entity)
```

### Step 9 — Module-detection pass

After Steps 1-8 complete, run `brownfield-engine/module-detector.ts` against `inventory.platforms.d365-ce` per [`templates/module-detection.yaml`](../module-detection.yaml). Detected modules emit module-specific artefact passes:

#### customer-service module artefacts

```
walk: Entities/sla/...                                      -> sla
walk: Entities/queue/...                                    -> queue
walk: Entities/routingrule/...                              -> routing-rule
walk: Entities/entitlement/...                              -> entitlement
walk: Entities/knowledgearticle/...                         -> kb-article
walk: Entities/msdyn_macro/...                              -> macro
walk: Entities/msdyn_productivityagentscript/...            -> productivity-pane
```

#### sales module artefacts

```
walk: Entities/{opportunity,quote,salesorder,invoice}/...   -> per-entity
walk: Entities/territory/...                                -> territory
walk: Entities/goal/...                                     -> goal
walk: Entities/pricelevel/...                               -> price-list
walk: Entities/product/...                                  -> product
```

#### marketing module artefacts

```
walk: Entities/msdynmkt_*/...                               -> per-entity
walk: Entities/list/...                                     -> marketing-list
walk: Entities/campaign/...                                 -> campaign
```

#### field-service module artefacts

```
walk: Entities/msdyn_workorder*/...                         -> per-entity
```

### Step 10 — Validate inventory coverage

Run `validate_inventory_coverage` against `inventory.platforms.d365-ce`. Any artifact category NOT present in expected list AND no instance is captured emits a gap-log entry of category `MISSING-INPUT` or `UNSUPPORTED-PATTERN`.

## Output schema

Conforms to [`schemas/brownfield-inventory.v1.json`](../../../schemas/brownfield-inventory.v1.json).
