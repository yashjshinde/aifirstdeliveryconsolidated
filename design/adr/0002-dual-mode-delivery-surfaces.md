# ADR-0002 — Dual-mode delivery surfaces

## Status

`accepted` — decided 2026-05-14.

## Context

The platform must serve two coexisting audiences without forcing a choice:

- **Single-agent users** who open just one agent folder in VS Code (e.g., `agents/d365-ce/`) and want to work there in isolation, without seeing the other seven agents.
- **Full-platform users** who open the repo root and want unified access to every agent's commands from one place, with namespace prefixes to disambiguate.

Both audiences also split by **tool**: Claude Code users invoke slash commands (`/spec`); GitHub Copilot (GHCP) users invoke chatmode + prompt files. The same agent author cannot be expected to write four parallel copies of every command — drift would be guaranteed.

## Decision

Generate **four delivery surfaces** from a **single authored source**:

| Surface | Where the user types | Where it lives | How it's produced |
|---|---|---|---|
| Claude standalone | `/spec` (no namespace) inside `agents/{a}/` | `agents/{a}/.claude/commands/spec.md` | **AUTHORED** — only source-of-truth |
| Claude root-unified | `/{agent}:spec` (namespaced) at repo root | `.claude/commands/{agent}/spec.md` | GENERATED by Publish Pipeline |
| GHCP standalone | `spec` prompt inside `agents/{a}/` | `agents/{a}/.github/prompts/spec.prompt.md` | GENERATED |
| GHCP root-unified | `{agent}-spec` prompt at repo root | `.github/prompts/{agent}-spec.prompt.md` | GENERATED |

**Authoring rule.** Every command is authored once in `agents/{a}/.claude/commands/*.md` (Claude-native location). The Publish Pipeline transforms it three times to produce the other three surfaces. See ADR-0003 for the source-of-truth rule and ADR-0011 for the 8-job pipeline that performs the rendering.

## Alternatives considered

- **Single surface (Claude root-only).** Reject — excludes GHCP users entirely and forces every user to open the repo root, breaking single-agent isolation.
- **Author per surface.** Reject — four parallel copies × 17 base commands × 8 agents = 544 hand-maintained files. Drift would be immediate and undetectable.
- **Live transformation at command time.** Reject — slow, opaque, and would require runtime tooling in both Claude and GHCP. Static pre-generated files are universally compatible.

## Consequences

**Positive:**
- One source of truth per command; transformation is mechanical and verifiable.
- Each agent folder works in isolation when opened directly in VS Code (Claude or GHCP).
- Root-level unified namespace supports multi-agent workflows (mixed-domain specs, aggregator commands, cross-agent handoffs).
- A drift-check CI job fails the build if any generated surface is hand-edited.

**Negative:**
- Build step required: the four surfaces are not authored — users editing a generated file see their change overwritten on next publish.
- Plugin distribution path (the truly portable form factor) lives behind the publish pipeline; folder-copy works only when the folder remains inside the consolidated repo (relative MCP path constraint — see ADR-0004).
