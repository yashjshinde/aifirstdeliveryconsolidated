### Module Architecture Summary

Configuration-only feature: extends msdyn_functionallocation Main form with hierarchy + 360 sub-grids. Aligns with Blueprint slide 22 (FS data model) and slide 23 (Site / FL ownership = Fusion).

**Data Flow:**
```
User → Functional Location form → hierarchy + sub-grid queries → Dataverse (Contacts / Assets / Cases / Work Orders / Invoices) → FL 360 view

Multi-language posture from day one (Constitution §15) means RESX-based string loading even though only `en-US` is initially shipped.
