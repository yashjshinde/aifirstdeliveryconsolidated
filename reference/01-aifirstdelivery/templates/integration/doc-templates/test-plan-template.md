## 1. Test Strategy

Risk-based: prioritise by data integrity risk and business criticality per interface.
Key risks: message loss, data corruption, duplicate processing, security breach, SLA breach.

## 9. Non-Functional Test Coverage

| NFR Category | Test Approach | Tool / Method | Acceptance Criteria |
|---|---|---|---|
| Performance | Sustained throughput and peak spike under load | Azure Load Testing | Per `constitution/11-nfr-targets.md` — API latency, async latency, throughput |
| Scalability | Concurrent message processing at {N}× normal volume | Azure Load Testing | No message loss, no throttle errors |
| Security | Auth, Managed Identity, Key Vault access, network isolation | Manual + code review | No secrets in code; Managed Identity enforced; private endpoints active |
| Reliability | DLQ, retry exhaustion, circuit breaker, idempotency | Fault injection + manual | No duplicate processing; DLQ populated on failure; alert fired |
| Availability | Service unavailability, network partition | Azure Chaos Studio | Graceful degradation; recovery within defined SLA |
