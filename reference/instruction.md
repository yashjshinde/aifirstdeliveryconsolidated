## Step 5 — What Claude does on resume

1. Reads `reference/00-spec-driven-development-humble-muffin.md` starting from the RESUME POINT
2. Confirms current state: R1–R33 applied; only port-scope question open
3. Offers the 3 port-scope options and waits for your decision:
   - **Option A** — Port BASIC agents first (solution-estimate + solution-architect); defer MATURE merges and Reporting/Brownfield (recommended)
   - **Option B** — Port all in one session
   - **Option C** — Defer all; document only

If Claude asks where reference files are, the resume prompt already gave the relative paths.

---

## Step 6 — Build order after port-scope decision

Regardless of which option you pick, each agent build follows this sequence:

1. **Constitution** — Author `agents/{agent}/constitution/0X-*.md`. PORT VERBATIM from `reference/01-aifirstdelivery/templates/{agent}/constitution/` where a source exists (F&O R16, brownfield R20, solution-estimate parts R32). Author fresh where no source exists.
2. **Templates** — Author `agents/{agent}/templates/`. PORTED with adaptation per the source-attribution tables in §7.X of the spec.
3. **Commands** — Author `agents/{agent}/.claude/commands/*.md` as the single canonical source (publish pipeline generates the 4 surfaces: standalone Claude / root-unified Claude / standalone GHCP / root-unified GHCP).
4. **Checklists** — Author `agents/{agent}/templates/checklists/*.checklist.md` (R16 pattern). PORT FO checklists verbatim; author fresh for other agents.
5. **README** — Author `agents/{agent}/README.md` in the What/How/Details structure (§21 of spec).
6. **Move to next agent.**
